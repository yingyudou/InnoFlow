# InnoFlow 创新开发辅助系统

> **📌 项目性质说明**  
> 本项目是一个**低保真模型（Low-fidelity Prototype）**，专为**科创课程**设计，用于展示创新管理系统的概念、功能和用户流程。本系统不连接真实后端服务，所有数据均为静态模拟数据，主要用于教学演示和用户流程验证。

## 🎯 项目简介

InnoFlow 是一个面向创新团队的辅助系统原型，旨在帮助用户按照科学的创新流程完成从"发现 → 定义 → 开发 → 交付"的全过程创新工作。作为低保真模型，本项目重点展示系统的核心功能流程和用户界面设计，为后续的系统开发提供概念验证和流程参考。

### 核心价值

- **流程可视化** - 清晰展示创新项目的完整生命周期
- **概念验证** - 验证创新管理系统的功能可行性和用户交互流程
- **教学演示** - 用于科创课程中的系统设计理念展示和用户体验测试

## 🔄 使用流程

### 1. 登录系统

- 进入 `idea.html` 查看 AI Idea 中心

#### 📋 定义阶段

- 文档管理和任务跟踪

#### ⚙️ 开发阶段 (Version Control)

- Canvas 绘制的版本树展示
- 多分支管理和版本历史查看
- 模拟版本回滚功能

#### 🚀 交付阶段
- 成果交付与推广应用
- 交付物管理
- AI 工具辅助

### 4. 辅助工具

#### 🤖 AI 会议助手 (`tool.html`)
- 上传会议录音文件（拖拽支持）
- 自动生成会议摘要
- 提取核心要点和待办事项
- 导出摘要功能

## 🚀 快速开始

### 运行方式

#### 方式一：直接打开（推荐用于快速预览）
1. 直接双击 `index.html` 文件
2. 在浏览器中查看系统界面

#### 方式二：本地服务器运行（推荐用于完整功能）
使用本地服务器可以避免一些浏览器安全限制：

```bash

python -m http.server 8000

# 使用 Node.js http-server
npx http-server -p 8000

# 使用 PHP
php -S localhost:8000
```

然后在浏览器中访问 `http://localhost:8000`

## ⚠️ 重要说明

### 低保真模型特性

本项目作为**低保真模型**，具有以下特点：

1. **静态数据** - 所有数据通过 JavaScript 对象模拟，刷新页面后数据会重置
2. **功能演示** - 部分功能仅展示 UI 和交互流程，未实现完整业务逻辑
3. **模拟算法** - AI 相关功能（聚类、摘要生成）为模拟实现，不调用真实 AI 服务
4. **无后端服务** - 不连接任何真实后端 API 或数据库
5. **演示目的** - 主要用于展示系统概念、验证用户流程和界面设计
