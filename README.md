# InnoFlow 创新开发辅助系统

> **📌 项目说明**  
> 本项目是专为**科创课程**设计的**创新开发辅助系统低保真模型（Low-fidelity Prototype）**。系统不连接真实后端服务，所有数据均为静态模拟数据，主要用于教学演示、概念验证和用户流程展示。

## 🎯 系统简介

InnoFlow 是一个面向创新团队的辅助系统原型，旨在帮助用户按照科学的创新流程完成从"发现 → 定义 → 开发 → 交付"的全过程创新工作。系统采用现代简洁的设计风格，通过可视化的方式展示创新项目的完整生命周期。

## ✨ 核心功能

### 1. 科学创新流程引导
系统将创新过程划分为四个清晰的阶段：
- **🔍 发现阶段** - 问题探索与机会识别，AI 驱动的 Idea 聚类与发散
- **📋 定义阶段** - 需求分析与方案设计，文档管理和任务跟踪
- **⚙️ 开发阶段** - 原型开发与迭代测试，分布式版本控制可视化
- **🚀 交付阶段** - 成果交付与推广应用，交付物管理

### 2. AI Idea 中心
- Canvas 绘制的气泡聚类可视化
- AI 智能聚类算法演示（模拟）
- Idea 详情查看和发散建议生成

### 3. 版本控制可视化
- 多分支结构展示
- 版本历史查看和详情展示
- 版本统计信息（提交次数、分支数、贡献者等）

### 4. AI 会议助手
- 文件上传功能（支持拖拽）
- 自动生成会议摘要（模拟）
- 待办事项提取
- 摘要导出功能

### 5. 项目与团队管理
- 项目列表展示和搜索
- 项目进度追踪
- 团队成员展示
- 项目负责人管理

## 🚀 快速开始

### 运行方式

#### 方式一：直接打开（推荐用于快速预览）
1. 直接双击 `index.html` 文件
2. 在浏览器中查看系统界面

#### 方式二：本地服务器运行（推荐用于完整功能）
使用本地服务器可以避免一些浏览器安全限制：

```bash
# 使用 Python 3
python -m http.server 8000

# 使用 Node.js http-server
npx http-server -p 8000

# 使用 PHP
php -S localhost:8000
```

然后在浏览器中访问 `http://localhost:8000`

### 登录说明
- 在登录页面输入**任意用户名和密码**即可进入系统
- 本低保真模型**无需真实验证**，仅用于流程演示

## 📖 使用教程

### 1. 登录系统
访问 `index.html` 进入登录页面，输入任意用户名和密码登录即可。

### 2. 查看项目列表
登录后进入项目主页（`dashboard.html`），可以看到所有创新项目卡片，每个卡片显示：
- 项目名称
- 当前阶段
- 进度百分比
- 负责人
- 更新时间

### 3. 进入项目详情
点击任意项目卡片，进入项目详情页（`project.html`），可以看到：
- 项目整体进度条
- 四个创新阶段的卡片，每个卡片显示：
  - 阶段进度
  - 阶段状态（已完成/进行中/未开始）
  - 阶段统计数据

### 4. 探索创新阶段

#### 发现阶段（Idea Center）
1. 点击"发现阶段"卡片进入 `idea.html`
2. 查看 Canvas 绘制的气泡聚类图
3. 点击气泡查看 Idea 详情和 AI 发散建议
4. 点击"AI 聚类"按钮触发聚类动画效果

#### 定义阶段
1. 点击"定义阶段"卡片进入 `define.html`
2. 查看需求文档列表
3. 查看任务跟踪和进度信息

#### 开发阶段（版本控制）
1. 点击"开发阶段"卡片进入 `version.html`
2. 查看版本统计信息
3. 选择不同分支查看版本树
4. 点击版本节点查看版本详情

#### 交付阶段
1. 点击"交付阶段"卡片进入 `deliver.html`
2. 查看交付物清单
3. 查看推广计划和应用场景

### 5. 使用 AI 工具
在项目详情页的"扩展工具"区域，可以访问：
- **AI 会议助手**（`tool.html`）
  - 上传会议录音文件（支持拖拽）
  - 点击"生成摘要"查看模拟生成的会议摘要
  - 查看核心要点、待办事项和创新建议
  - 导出摘要或重新上传

## ⚠️ 重要说明

### 低保真模型特性

本项目作为**低保真模型**，具有以下特点：

1. **静态数据** - 所有数据通过 JavaScript 对象模拟，刷新页面后数据会重置
2. **功能演示** - 部分功能仅展示 UI 和交互流程，未实现完整业务逻辑
3. **模拟算法** - AI 相关功能（聚类、摘要生成）为模拟实现，不调用真实 AI 服务
4. **无后端服务** - 不连接任何真实后端 API 或数据库
5. **演示目的** - 主要用于展示系统概念、验证用户流程和界面设计

### 适用场景
- ✅ 科创课程教学演示
- ✅ 用户流程验证和测试
- ✅ 系统概念展示
- ✅ 界面设计参考

### 不适用场景

- ❌ 真实用户使用

- ❌ 真实 AI 功能调用
